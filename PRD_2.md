# PRD: Система учёта оборудования и имущества сотрудников

**Версия:** 1.0
**Дата:** 07.07.2026
**Статус:** Черновик
**Автор:** —

---

## 1. Цель и контекст

Компании необходим простой инструмент для учёта:
1. Оборудования организации и его списания.
2. Имущества (техники, спецодежды, обуви, СИЗ), выданного сотрудникам, с учётом их размеров.

Сейчас учёт ведётся вручную/разрозненно, из-за чего:
- Сложно понять, что и в каком состоянии находится на балансе.
- Списание оборудования не фиксируется системно, нет истории и причин.
- Неизвестно, что именно выдано конкретному сотруднику и когда пора менять изношенные вещи.
- При выдаче одежды/обуви приходится каждый раз уточнять размер сотрудника.

**Решение первой итерации:** два связанных Excel/Google Sheets реестра с автоматизацией через формулы, выпадающие списки и условное форматирование — без необходимости в отдельном ПО или базе данных.

---

## 2. Проблема пользователя

| Роль | Проблема |
|---|---|
| Завхоз / офис-менеджер | Не видит актуальный статус оборудования (в эксплуатации / на списание / списано), нет истории списаний |
| Кадровик / кладовщик | Не знает, что выдано сотруднику, какого размера одежда/обувь, когда истекает срок носки |
| Бухгалтер | Нет данных для расчёта стоимости списанного имущества |
| Сотрудник | Долго ждёт выдачи, т.к. размер каждый раз уточняется заново |

---

## 3. Цели и метрики успеха

**Цели:**
- Все оборудование и выданное имущество зафиксированы в единой структуре.
- Статус и сроки видны сразу, без ручного анализа.
- Размеры сотрудников не нужно искать/спрашивать повторно.
- Списание и выдача сопровождаются документом для печати/подписи.

**Метрики успеха:**
- 100% оборудования внесено в реестр с актуальным статусом.
- Время подготовки акта/ведомости сокращено с "искать вручную" до "открыть готовый лист".
- 0 просроченных по сроку носки позиций остаются незамеченными (все подсвечены).

## 4. Область продукта (Scope)

### 4.1 Модуль 1 — Реестр оборудования и списания
Файл: `reestr_oborudovaniya.xlsx`

**Функции:**
- Реестр единиц оборудования: инв. номер, наименование, категория, подразделение, ответственный, дата постановки на учёт, стоимость, срок службы.
- Поля "Состояние" (Исправно / Требует ремонта / Неисправно) и "Статус" (В эксплуатации / На ремонте / На списание / Списано) — выпадающие списки.
- Причина и дата списания.
- Условное форматирование строк по статусу.
- Лист "Сводка": количество единиц по статусам, итого, суммарная стоимость списанного оборудования.

### 4.2 Модуль 2 — Имущество и размеры сотрудников
Файл: `imushestvo_sotrudnikov.xlsx`

**Функции:**
- Справочник сотрудников: ФИО, табельный номер, подразделение, должность, размер одежды, размер обуви, рост, контакты.
- Реестр выдачи: ФИО (из справочника), категория (Техника / Спецодежда / Обувь / СИЗ / Другое), наименование, инв. номер / размер, количество, дата выдачи.
- Автоподстановка размера одежды/обуви сотрудника при выборе ФИО (VLOOKUP).
- Срок носки (мес.) → автоматический расчёт даты плановой замены и статуса ("Пора менять" / "ОК") с подсветкой.
- Статус выдачи (Выдано / Возвращено / Списано / На замену) с подсветкой.
- Лист "Сводка": количество по категориям, число сотрудников, число позиций "на замену" и "пора менять".
- Лист "Ведомость на выдачу": печатная форма 1:1 с реестром, с автоподстановкой должности, полем для подписи, альбомная ориентация под печать.

### 4.3 Вне текущей области (возможные следующие итерации)
- Объединение двух файлов в одну систему с общим справочником оборудования.
- Полноценный акт списания как отдельный печатный документ с нумерацией и печатью.
- Уведомления (email/Telegram) при приближении срока замены.
- Web/Telegram-бот интерфейс вместо таблиц (если потребуется многопользовательская работа в реальном времени).
- Права доступа / разграничение по подразделениям.
- Штрихкоды/QR для инвентарных номеров.

## 5. Пользовательские сценарии (User Stories)

1. **Как завхоз**, я хочу видеть все оборудование со статусами, чтобы понимать, что пора списать.
2. **Как завхоз**, я хочу фиксировать причину и дату списания, чтобы иметь историю для проверок.
3. **Как кадровик**, я хочу one-click видеть размер сотрудника при выдаче одежды, чтобы не тратить время на уточнения.
4. **Как кладовщик**, я хочу получать сигнал, когда истекает срок носки СИЗ/одежды, чтобы вовремя организовать замену.
5. **Как бухгалтер**, я хочу видеть сумму списанного оборудования без ручного подсчёта.
6. **Как HR**, я хочу распечатать ведомость с подписью сотрудника при выдаче имущества.

## 6. Требования к реализации

### Функциональные требования
- FR1: Выпадающие списки для всех статусных полей (без свободного ввода текста).
- FR2: Условное форматирование по цвету статуса/срока.
- FR3: Формулы для сводных показателей (COUNTIF/SUMIF), без хардкода значений.
- FR4: Связь между листами через VLOOKUP (единый источник правды — справочник сотрудников).
- FR5: Ноль ошибок формул (#REF!, #DIV/0! и т.д.) при пересчёте.
- FR6: Печатная форма с фиксированной шапкой и альбомной ориентацией.

### Нефункциональные требования
- NFR1: Совместимость с Excel и Google Sheets (импорт xlsx).
- NFR2: Понятный интерфейс без обучения — заголовки на русском, цветовая индикация.
- NFR3: Расширяемость — до 60–80 строк на лист без потери производительности, легко увеличить.
